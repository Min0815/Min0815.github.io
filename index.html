요구사항 정의

사업 목표를 달성하기 위해 총 23개의 요구사항을 8개 카테고리로 체계적으로 정의하고 문서화했습니다. 각 요구사항은 고유 번호·정의·세부 내용·제안 내용 구조로 작성하여 추적 가능성을 확보했습니다.

Bastion: t2.micro (2vCPU / 1GiB) · App·DB: m5.large (2vCPU / 8GiB)

🔁 고가용성 웹 인프라
ALB → Auto Scaling Group 구성으로 트래픽 급증 시 EC2 자동 증설. Health Check 기반 장애 서버 자동 교체. WAS + DB 분리된 멀티 티어 구성. 두 개의 가용 영역(AZ-A / AZ-B)에 분산 배치하여 단일 AZ 장애를 허용합니다.
☁️ 서버리스 문의 시스템
S3 정적 페이지 호스팅 + CloudFront CDN 캐싱·가속. Lambda + SNS 이벤트 기반 처리로 상시 EC2 운영 없이 과금 효율화. 요청이 있을 때만 과금되는 구조로 운영 비용을 절감합니다.
🛡️ 보안 다층 방어
WAF: SQL Injection·XSS 차단 (OWASP Managed Rule Set). ACM: SSL/TLS 인증서 암호화. Session Manager: SSH 포트 미개방. Private Subnet: WAS·DB 외부 미노출. IAM 최소 권한 원칙 적용.
📊 로그인 서비스 흐름
고객 브라우저 접속 → Route 53 DNS 쿼리 → WAF 공격 차단 → CloudFront 캐싱 전송 → ALB 로드밸런싱 → WAS 인증 처리 → DB 계정 대조. 세션은 Redis 또는 RDS 기반으로 유지합니다.

06

향후 계획

1
DR 자동화 파이프라인 구축
재해 복구 시나리오 자동화
Lambda + CloudWatch Events로 RDS 스냅샷 생성·교차 리전 복사(Cross-Region Replication)를 스케줄 기반으로 자동화합니다.
장애 감지 시 다른 리전에서 RDS 인스턴스를 자동 복원하고, Route 53 DNS 레코드 변경으로 트래픽을 자동 전환하는 DR 워크플로우를 구축합니다.
RTO(목표 복구 시간) 및 RPO(목표 복구 시점)를 정량 목표로 설정하고, 분기별 모의 훈련을 통해 워크플로우를 검증합니다.
2
GCP 권한 기반 접근 제어
계정별 RAG 접근 권한 차등 적용
현재 Google Drive ACL을 Kendra에서 상속하는 방식에서 나아가, 사용자별 조직 계정 정보를 Lambda에서 검증한 후 Kendra 쿼리 필터에 반영합니다.
직급·팀 단위 문서 접근 정책을 Kendra의 Access Control List(ACL)와 연동하여 민감 문서가 권한 없는 직원에게 노출되지 않도록 합니다.
AWS IAM Identity Center와 SSO를 연계하여 사내 시스템 로그인을 단일화하고 권한 관리 오버헤드를 줄입니다.
3
데이터 보존·삭제 자동화
규제 대응 및 비용 최적화
GDPR·국내 개인정보보호법 변화에 맞춰 데이터 삭제·익명화 절차를 자동화하는 솔루션(AWS Macie + Lambda) 도입을 검토합니다.
S3 Intelligent-Tiering을 적용하여 접근 빈도에 따라 스토리지 클래스를 자동 조정하고, Glacier Instant Retrieval로 아카이빙 비용을 절감합니다.
데이터 파이프라인(Kinesis → Glue ETL)에 유효성 검사 모듈을 추가하고, 무결성 위반 감지 시 SNS 알림 및 초기 복구 워크플로우를 자동 트리거합니다.
4
AI 답변 품질 지속 개선
사용자 피드백 루프 고도화
Q&A 인터페이스에 👍 / 👎 피드백 버튼을 추가하고, DynamoDB에 피드백 데이터를 축적하여 답변 품질을 정량적으로 추적합니다.
피드백 점수가 낮은 질의 유형을 분석해 Kendra 인덱스의 Relevance Tuning과 Bedrock 시스템 프롬프트를 개선하는 주기적 리뷰 프로세스를 도입합니다.
Kendra 동기화 주기를 트래픽 패턴에 따라 가변 조정하고, Bedrock 온디맨드 토큰 과금 최적화를 통해 AI 서비스 운영 비용을 단계적으로 절감합니다.

프로젝트 회고

01
인프라 설계의 비용 최적화 관점
Lambda + S3 서버리스 구성을 통해 상시 EC2 운영 대비 비용을 절감했습니다. 서비스 요구사항에 맞는 아키텍처 선택이 단순한 기술 도입을 넘어 비용 구조에 직접 영향을 미친다는 점을 실감했습니다.
